DPUB footnote display test

footnote-display: block (default)

Each semantic footnote body should start on its own line in the footnote area by default.

Block footnotes remain separated1 even when the note text is short2 and could otherwise fit beside another note3.

footnote-display: inline

Inline footnotes should stay in one inline run even when a longer note wraps, with a normal space between note bodies.

Inline footnotes should begin as a loose run of notes1 and then continue through a longer inline note that wraps to the next line in the footnote area2 before a trailing short note carries on inline rather than becoming a separate block3.

footnote-display: compact

Compact footnotes should behave like unbreakable inline notes by default, and only fall back to block when a single note cannot fit on one rendered line.

Four short compact notes should wrap as inline nowrap units1 with the next short compact note still on that first line2 and then continue on the next line with another short compact note3 plus one more short compact note on that same following line4.

A compact note that is too long to fit as a single unbroken inline note should fall back to block5 while the following short compact notes can still share a later line inline6 and continue with another short compact note on that same line7 before a final longer compact note falls back to block again8.

footnote-display: inline with outside markers

Outside footnote markers should keep each semantic footnote as a list item, even when footnote-display:inline is specified.

Outside-marker inline footnotes should not collapse into one inline run1 when the marker is rendered outside the footnote body2 and a third footnote should still remain a separate list item3.

Outside markers should also prevent compact semantic footnotes from being forced into inline nowrap rendering.

Compact outside-marker footnotes should stay as separate list items1 rather than sharing a single inline row2 even when another short footnote follows3.

footnote-display: inline with author ::after separator

When author CSS sets .footnote::after content, that content should replace the default space separator between inline footnotes, and line breaks should still be allowed between adjacent footnotes even though the author separator is not whitespace.

Author CSS should be able to use a visible non-space separator between inline semantic footnotes1 and keep using that author-defined separator for the next inline footnote2 instead of the default single ASCII space3.

1AuthorSeparatorAlphaSegment.
2AuthorSeparatorBetaSegment.
3AuthorSeparatorGammaSegment.

footnote-display: compact with author ::after none

When author CSS sets .footnote::after to none, compact inline footnotes should appear with no default separator inserted between them, but adjacent compact notes should still be able to wrap between notes.

Compact semantic footnotes should allow the author to suppress separators entirely1 so the next compact footnote follows with no generated gap yet can still move to the next line2 and a third note also keeps that explicit no-separator behavior3.

1CompactBreakAlphaSegment.
2CompactBreakBetaSegment.
3CompactBreakGammaSegment.
